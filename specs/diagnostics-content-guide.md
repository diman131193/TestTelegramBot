# Как редактировать диагностику

Основной редактор контента — `content/diagnostics-content.xlsx`. Заказчику не нужно редактировать JSON или Python-код.

## Цвета в таблице

- желтые ячейки — тексты и настройки, доступные для редактирования;
- серые ячейки — служебные ID и переходы, которые меняются разработчиком;
- зеленые строки — заголовки таблиц.

## Листы

### `Рекомендации`

Каждая строка — самостоятельная карточка.

Роли:

- `Основная` — главный вывод; в результат попадает ровно одна карточка;
- `Стоп-флаг` — важное ограничение; попадают все актуальные карточки;
- `Дополнение` — полезный контекст; попадает не более двух карточек.

Заказчик заполняет ситуацию, заголовок, объяснение, практические шаги, ограничения и CTA.

Активная карточка должна иметь статус `Готово`. Для незавершенной работы нужно выбрать `Черновик` и `Активно = Нет`.

### `Вопросы`

Можно менять формулировки вопросов и ответов. ID и следующий вопрос являются частью логики и меняются разработчиком.

### `Подписи`

Здесь настраиваются короткие фразы раздела «Что я учла».

### `Общие тексты`

Содержит заголовки секций, общий дисклеймер и ограничения количества блоков.

### `Услуги`

Содержит актуальный каталог услуг, показания, ограничения, CTA и ссылку записи. Ограничения должны быть подтверждены мастером.

### `Предпросмотр`

Позволяет выбрать несколько карточек и посмотреть их тексты рядом. Полный результат по конкретной цепочке ответов показывает CLI-команда `preview`.

## Проверка

```bash
python scripts/manage_diagnostics_content.py validate
```

Проверка выполняет следующее:

- проверяет обязательные поля и ID;
- проверяет переходы между вопросами;
- проверяет связи правил и карточек;
- проходит все 3 696 допустимых путей;
- проверяет наличие основного результата;
- проверяет лимит сообщения Telegram.

## Предпросмотр результата

```bash
python scripts/manage_diagnostics_content.py preview \
  --answers 1.4,2.3,3.4,4.2,6.2
```

Если `--answers` не передан, используется встроенный пример.

## Публикация

```bash
python scripts/manage_diagnostics_content.py publish
```

Публикация выполняется атомарно и только после успешной проверки. Вопросы и контент записываются одним согласованным снимком в `data/diagnostics_snapshot.json`.

Новые прохождения используют опубликованный контент без перезапуска бота.

## Что остается технической настройкой

- `data/diagnostic_factors.json` — смысл ответов и внутренние признаки;
- `data/diagnostic_rules.json` — условия выбора основных модулей, дополнений и стоп-флагов.

Эти файлы редактирует разработчик. Внутренние числовые признаки пользователю не показываются.
