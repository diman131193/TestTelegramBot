# Обзор продукта

## Назначение

`TestTelegramBot` — Telegram-бот для коммуникации мастера по реконструкции и восстановлению волос с двумя аудиториями:

- клиенты, которым нужны услуги, отзывы, запись, консультация и диагностика состояния волос;
- мастера, которым в будущем может быть показана обучающая или профессиональная информация.

## Цели

- Быстро объяснить пользователю, кто такой мастер и чем может помочь.
- Разделить поток клиентов и мастеров уже на старте.
- Дать клиенту понятную навигацию по услугам.
- Показать отзывы и результаты.
- Перевести пользователя к записи через внешнюю ссылку.
- Собрать вопрос на консультацию и передать его администратору.
- Провести бесплатную диагностику волос и собрать персональную рекомендацию из основного вывода, стоп-флагов и контекстных дополнений.
- Логировать активность пользователей в локальную SQLite-базу.

## Роли

### Клиент

Пользователь, который пришел узнать об услугах, посмотреть отзывы, записаться или задать вопрос мастеру.

### Мастер

Пользователь, выбравший ветку `Мастер`. В текущей версии получает заглушку `Скоро здесь появится информация`.

### Администратор

Telegram-чат с ID из переменной окружения `ADMIN_CHAT_ID`. Получает вопросы клиентов и может ответить клиенту через reply на сообщение бота.

## Главные сценарии

1. Пользователь отправляет `/start`.
2. Бот показывает приветствие с фото и выбором роли.
3. Клиент выбирает `Клиент`.
4. Бот показывает клиентское меню: услуги, прайс, отзывы, диагностика, запись, консультация.
5. Клиент выбирает услугу и получает описание с фото.
6. Клиент смотрит отзывы или переходит на запись.
7. Клиент запускает диагностику, отвечает на вопросы и получает рекомендацию.
8. Клиент выбирает консультацию, пишет вопрос, бот пересылает его администратору.
9. Администратор отвечает reply-сообщением, бот отправляет ответ клиенту.

## Метрики качества

- Все кнопки, показанные пользователю, должны иметь ожидаемое поведение.
- Пользователь не должен попадать в молчаливый сценарий после нажатия кнопки.
- Тексты и медиа должны загружаться из JSON без падения бота.
- Вопросы консультации должны доходить до администратора.
- Ответ администратора должен приходить конкретному клиенту.
- Диагностика должна завершаться для любой валидной комбинации ответов.
- Бот должен логировать последнее действие пользователя.
